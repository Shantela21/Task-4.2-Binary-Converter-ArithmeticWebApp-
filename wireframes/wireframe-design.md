# Binary Converter & Arithmetic Web App - Wireframe Design

## Layout Overview

The app features a clean, modern interface with four main sections accessible via tab navigation:

### Header Section
- **App Title**: "Binary Converter & Arithmetic Web App"
- Centered, prominent typography
- Clean, professional appearance

### Tab Navigation
- **4 Tabs**: Calculator | Converter | BODMAS | ASCII Table
- Horizontal tab bar with active state indicators
- Responsive design that stacks on mobile

### Tab Content Areas

#### 1. Calculator Tab
- **Input Fields**: 
  - "Value 1" (binary input)
  - "Value 2" (binary input)
- **Operation Buttons**: +, −, ×, ÷ in grid layout
- **Calculate Button**: Prominent, full-width
- **Result Display**: Clear result area with both binary and decimal

#### 2. Converter Tab
- **Decimal to Binary Section**:
  - Decimal input field
  - Convert button
  - Result display
- **Binary to Decimal Section**:
  - Binary input field  
  - Convert button
  - Result display

#### 3. BODMAS Tab
- **Expression Input**: Large text area for binary expressions
- **Example placeholder**: "(1010 + 0101) * 10"
- **Evaluate Button**
- **Result Display**: Shows binary and decimal results

#### 4. ASCII Table Tab
- **Responsive Table** with columns:
  - Symbol (character)
  - Binary Code (8-bit)
  - HTML Number (entity)
  - Description
- **Scrollable** for mobile devices
- **Hover effects** on rows

### Footer
- Copyright information
- Simple, unobtrusive design

## Design Principles

### Color Scheme
- **Primary**: Blue (#2563eb) for main actions and active states
- **Success**: Green (#22c55e) for successful conversions
- **Error**: Red (#ef4444) for validation errors
- **Background**: Light gray (#f8fafc) for main background
- **Surface**: White (#ffffff) for cards and inputs

### Typography
- **Font**: System font stack for optimal performance
- **Hierarchy**: Clear distinction between headers, labels, and body text
- **Weights**: 600 for labels, 700 for headers, normal for body

### Spacing
- **Consistent**: 1rem base unit for spacing
- **Breathing room**: Adequate padding between elements
- **Groups**: Related elements grouped visually

### Responsive Behavior
- **Desktop**: 800px max width, centered layout
- **Tablet**: Adjusted margins and padding
- **Mobile**: Stacked tabs, simplified button layouts

## Accessibility Features
- **Semantic HTML**: Proper use of header, main, section, nav
- **Focus states**: Clear keyboard navigation
- **Color contrast**: WCAG compliant color ratios
- **Reduced motion**: Respects user preferences
- **Screen reader**: Proper labeling and descriptions

## User Experience Flow
1. User lands on Calculator tab (default)
2. Can switch between tabs seamlessly
3. Input validation provides immediate feedback
4. Results displayed clearly with both binary and decimal
5. Error messages are helpful and specific
6. Mobile-friendly touch targets

This wireframe emphasizes usability, accessibility, and modern design principles while maintaining simplicity and focus on the core functionality.
